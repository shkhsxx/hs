## Case 01

**유형:** Slack DM

**기능명:** 로그인 후 대시보드 자동 진입

**기능 목적:** 로그인한 유저가 대시보드를 즉시 확인할 수 있도록 함

### 확정 요구사항
- 로그인 완료 시 대시보드 화면을 즉시 표시
- 대시보드에 매출 그래프 포함
- 매출 그래프는 월별 단위로 표시
- 미인증 유저가 대시보드 URL 직접 접근 시 로그인 페이지로 리다이렉트

### 의견/제안
- 기간 필터: "나중에 해도 될 것 같아요" → 확정 아님
- 주별 그래프: "주별은 다음에~" → 확정 아님

### 정책
- **로그인 정책:** 미인증 상태 대시보드 접근 불가, 로그인 페이지로 강제 리다이렉트
- **권한 정책:** 로그인 유저만 대시보드 접근 허용

### 수치
- 없음

### 예외 상황
- 미인증 유저의 대시보드 직접 URL 접근 → 로그인 페이지 리다이렉트

### Priority
**P1** — "대시보드 바로 뜨게 해주세요" (필수 표현)

## Case 06

**유형:** Slack Channel Thread

**기능명:** 결제 실패 자동 재시도 및 구독 정지 처리

**기능 목적:** 결제 실패 시 자동 재시도로 구독 유지율 제고

### 확정 요구사항
- 결제 실패 시 자동으로 최대 3회 재시도
- 재시도 간격: 1일 1회 (총 3일)
- 3회 모두 실패 시 구독 정지 (데이터는 삭제하지 않고 보관)
- 결제 수단(카드) 변경 시 즉시 재시도
- 알림 발송: 첫 번째 실패 시 + 마지막(3번째) 실패 시만 발송 (중간 실패 알림 없음)

### 의견/제안
- 없음 (모든 요구사항 확정)

### 정책
- **결제 정책:** 실패 시 최대 3회 재시도, 간격 24시간
- **구독 정책:** 3회 실패 시 구독 정지, 데이터 보관
- **알림 정책:** 결제 실패 알림은 1회차, 3회차만 발송
- **저장 정책:** 구독 정지 후에도 유저 데이터 삭제하지 않음

### 수치
- 재시도 횟수: 3회
- 재시도 간격: 1일 (하루 1회)
- 알림 발송 시점: 1회차 실패, 3회차 실패

### 예외 상황
- 3회 모두 실패 → 구독 정지 (데이터 보관)
- 재시도 도중 카드 변경 → 즉시 재시도
- 구독 정지 후 결제 재시도 가능 여부 및 복구 프로세스 — 원문 언급 없음

### Priority
**P1** — "결제 실패하면 자동으로" (직접적 기능 구현 요청, 구독 서비스 핵심 정책)
